# ExcelEncryptor

ExcelEncryptor is a small, focused, commercially free .NET library for applying Excel-compatible Agile Encryption to OOXML workbooks.

It is intended to encrypt `.xlsx` and `.xlsm` files generated by libraries such as ClosedXML, NPOI, and DocumentFormat.OpenXml.

> Inspired by [Apache POI](https://github.com/apache/poi)'s Agile encryption implementation.

## Features

- AES-128 / 192 / 256 encryption
- Multiple hash algorithms: MD5, SHA-1, SHA-256, SHA-384, SHA-512
- Compatible with Microsoft Excel password-protected files
- Cross-compatible with Java POI encrypted files
- No dependency on NPOI or ClosedXML — works with any xlsx source

## File format support

- Supported: `.xlsx`, `.xlsm`
- Not supported: `.xls`, `.xlsb`

## Security defaults

- Recommended for new encryption: `Aes256` + `Sha512`
- Compatibility-only legacy options: `Sha1`, `Md5`
- `Md5` is supported only for compatibility and is not recommended for new files
- Security scope and limitations: [docs/security.md](docs/security.md)
- Compatibility matrix and limits: [docs/compatibility.md](docs/compatibility.md)

## Usage

### Minimal example

```csharp
using ExcelEncryptor;

Encrypt.FromFileToFile("input.xlsx", "encrypted.xlsx", "pass");
var plainBytes = Encrypt.Decrypt("encrypted.xlsx", "pass");
```

### ClosedXML workbook example

```csharp
using ClosedXML.Excel;
using ExcelEncryptor;

using var wb = new XLWorkbook();
wb.AddWorksheet("Sheet1").Cell("A1").Value = "Hello";
using var ms = new MemoryStream();
wb.SaveAs(ms);

Encrypt.FromBytesToFile(ms.ToArray(), "closedxml.xlsx", "pass");
```

### NPOI workbook example

```csharp
using ExcelEncryptor;
using NPOI.XSSF.UserModel;

var wb = new XSSFWorkbook();
wb.CreateSheet("Sheet1").CreateRow(0).CreateCell(0).SetCellValue("Hello");

using var ms = new MemoryStream();
wb.Write(ms, leaveOpen: true);
Encrypt.FromBytesToFile(ms.ToArray(), "npoi.xlsx", "pass");
```

### OpenXML SDK workbook example

```csharp
using DocumentFormat.OpenXml.Packaging;
using DocumentFormat.OpenXml.Spreadsheet;
using ExcelEncryptor;

using var ms = new MemoryStream();
using (var doc = SpreadsheetDocument.Create(ms, DocumentFormat.OpenXml.SpreadsheetDocumentType.Workbook, true))
{
    var workbookPart = doc.AddWorkbookPart();
    workbookPart.Workbook = new Workbook(new Sheets());
}

Encrypt.FromBytesToFile(ms.ToArray(), "openxml.xlsx", "pass");
```

For full API documentation, see `src/ExcelEncryptor/README.md`.

## Testing

Run the test suite to verify encryption/decryption compatibility and regenerate the manual-test artifacts:

```bash
dotnet test
```

This generates encrypted workbooks in `test-manual-files/`:

- `simple_en.xlsx`, `simple_ja.xlsx`
- `japanese_en.xlsx`, `japanese_ja.xlsx`
- `excel_en.xlsm`, `excel_ja.xlsm`
- `excel_image_en.xlsx`, `excel_image_ja.xlsx`

If you want to refresh only those manual artifacts, run:

```bash
dotnet test tests/ExcelEncryptor.Tests/ExcelEncryptor.Tests.csproj --filter Generate_ManualTestFiles_WritesEncryptedWorkbooks
```

Validated workbook sizes in automated tests: `1 MB`, `10 MB`, `50 MB`, `100 MB`. Encrypt/decrypt elapsed times are printed to test output.
